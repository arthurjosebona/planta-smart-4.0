Etapa 1 · Regras de Negócio domínio⏱ 02:00

Regras que o modelo precisa garantir

  • ▸
    Pedido triplo exige exatamente 3 blocos — simples = 1, duplo = 2, triplo = 3. O tipo precisa bater com a quantidade.
  • ▸
    Vínculos exclusivos — bloco pertence a um único pedido; lâmina pertence a um único bloco.
  • ▸
    Estoque com 28 posições — cada uma com uma cor; consumida ao montar um bloco.
  • ▸
    Expedição com 12 posições — pedido concluído ocupa uma vaga de saída.
  • ▸
    Sem dois blocos no mesmo andar — cada andar do pedido é único.

Controller → Service → Repository

Etapa 2 · Camada Service filtro de qualidade⏱ 05:15

Validações reais de negócio

  • ✓
    Campos obrigatórios — blocos, tipo, cor da tampa  → RequiredFieldException
  • ✓
    Tipo bate com a quantidade de blocos → TipoIncompativelComBlocosException
  • ✓
    Ordem de produção não se repete → OrdemDeProducaoExistenteException
  • ✓
    Nenhum andar duplicado → DuplicateAndarException
  • ✓
    Vínculo automático com estoque — busca posição com a cor certa e marca como vazia  → EstoqueInsuficienteException
  • ✓
    Conclusão associa à expedição — status CONCLUIDO + vaga de saída  → ExpedicaoLotadaException
Etapa 2 · Tratamento de Erros destaque técnico⏱ 06:00

if (repo.existsByOrdemDeProducao(op))
    throw new OrdemDeProducaoExistenteException();
