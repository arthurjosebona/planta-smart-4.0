Abertura · Contexto

O que é a Bancada Smart 4.0

É um sistema que controla, pelo computador, uma linha de produção física em miniatura — a bancada didática do laboratório. Ela tem quatro estações automatizadas, cada uma com seu próprio CLP Siemens.

📦

Estoque

Armazenamento e gestão das peças disponíveis para produção.

⚙️

Processo

Etapa de processamento das peças.

🤖

Montagem

Montagem dos componentes do produto final.

🚚

Expedição

Saída e rastreamento dos pedidos concluídos.

⚡ O que é um CLP? Controlador Lógico Programável — um computador industrial robusto, feito pra ligar/desligar motores e sensores. É o "cérebro burro" da máquina: executa comandos simples, mas alguém precisa organizar as informações. É aí que entra o nosso sistema.
Abertura · Motivação por que existe

O problema que o sistema resolve

  • ▸
    A bancada sozinha só executa — não tem interface amigável, não guarda histórico, não sabe o que é um "pedido".
  • ▸
    O Smart 4.0 dá um cérebro de gestão — cria pedidos de produção, controla o estoque de peças, acompanha a expedição.
  • ▸
    Tudo em tempo real, numa tela só — sem precisar abrir softwares industriais especializados (como o TIA Portal).

Modelagem · Regras de negócio domínio

Regras que o sistema precisa garantir

  • ✓
    Campos obrigatórios não podem faltar — blocos, tipo, cor da tampa → RequiredFieldException
  • ✓
    Tipo precisa bater com a quantidade de blocos — simples = 1, duplo = 2, triplo = 3 → TipoIncompativelComBlocosException
  • ✓
    Ordem de produção não se repete → OrdemDeProducaoExistenteException
  • ✓
    Sem dois blocos no mesmo andar — cada andar do pedido é único → DuplicateAndarException
  • ✓
    Estoque precisa ter posição disponível da cor certa → EstoqueInsuficienteException
  • ✓
    Expedição não passa de 12 posições ocupadas → ExpedicaoLotadaException
  • ✓
    Só inicia produção se o pedido estiver pendente → PedidoNaoPendenteException

Conceito · API REST como front e back conversam

O que é uma API REST

Frontend e backend são programas separados, em linguagens diferentes. Para conversarem, combinam um idioma comum: HTTP. Uma API REST é um conjunto de endereços — cada verbo tem um significado combinado.

GET/api/pedidosme dá a lista de pedidos
POST/api/pedidoscrie um pedido novo
PUT/api/pedidos/start-production/{id}inicia a produção
PUT/api/pedidos/{id}atualize esse pedido
DELETE/api/pedidos/{id}apague esse pedido
📖 Swagger / OpenAPI Documentamos todos os endpoints numa página interativa — dá pra ver e testar cada rota direto pelo navegador, sem escrever código.
Conceito · Arquitetura em camadas backend

Controller · Service · Repository

Dentro do backend, cada camada só conhece a camada logo abaixo. Isso facilita trocar uma peça sem quebrar o resto.

🎮 ControllerPorta de entrada REST — recebe a requisição HTTP e devolve a resposta.entrada
⚙️ ServiceOnde mora a regra de negócio — o "filtro de qualidade" do sistema.negócio
🗃️ RepositoryAcesso ao banco de dados, sem SQL escrito à mão.dados
🧱 Model / EntityEstrutura dos dados — Pedido, Bloco, Lâmina, Estoque, Expedição.domínio
📤 DTOFormato dos dados que trafegam entre frontend e backend.resposta

📦

🚚

🗃️ JPA/Hibernate Em vez de escrever SQL à mão, ele transforma objetos Java (como a classe Pedido) direto em linhas de tabela, e vice-versa.
Conceito · Integração física a parte mais avançada

Comunicação com o CLP — protocolo S7

Os CLPs Siemens usam o protocolo S7, sobre TCP/IP, porta 102. Normalmente isso exige um software pronto (como o TIA Portal).

🔌

Implementado do zero

Sem bibliotecas prontas de terceiros — em Java puro.

↔️

Bidirecional

O backend lê o status das estações e também escreve comandos.

🧵

Multi-thread

Conexões concorrentes, uma leitura ativa por estação monitorada.

⚙️ Baixo nível Traduzimos bytes crus (blocos de dados / DBs) em informação que o sistema entende, tipo "estação Estoque está ocupada" ou "posição 3 tem uma peça azul".
Conceito · Tempo real Server-Sent Events

A tela que se atualiza sozinha

Numa requisição normal, o navegador pergunta e espera resposta. Mas a tela precisava se atualizar sozinha conforme o status da bancada muda.

📡 SSE O navegador abre uma conexão e o servidor vai empurrando atualizações conforme acontecem — sem o frontend ficar perguntando toda hora.
📰 Analogia É como assinar um canal de notícias: você não recarrega a página pra saber a novidade, ela chega sozinha.
GET/api/smart/streamSSE multiplexado com eventos de todas as estações
Conceito · Frontend React + TypeScript

Atomic Design e MVVM

Usamos React (componentes reutilizáveis) com TypeScript, que adiciona tipos e avisa erros antes mesmo de rodar o código.

🧩

Atomic Design

Organiza componentes por complexidade: átomos (botão, badge) → moléculas (card) → organismos (uma seção completa da tela).

🔁

MVVM

Model define os dados · View só desenha a tela · ViewModel guarda estado e lógica de interação num hook.

Conceito · Deploy Docker

Rodar tudo junto, sem dor de cabeça

O sistema tem três partes que precisam rodar juntas: banco, backend e frontend. Instalar tudo manualmente em cada máquina gera o clássico "na minha máquina funciona".

🐳 Container Cada parte do sistema é empacotada num ambiente isolado e padronizado, com tudo que precisa pra rodar.
▶️ Um comando docker compose up sobe os três serviços já configurados e conectados entre si.
Conceito · Qualidade testes automatizados

Confiança pra mudar o código

  • ✓
    Testes unitários — testam uma peça isolada, ex: uma regra do PedidoService sozinha, sem banco real.
  • ✓
    Testes de integração — testam o fluxo completo (criar um pedido e verificar se refletiu no estoque), usando um banco em memória (H2) só durante o teste.
